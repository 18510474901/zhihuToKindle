Wordpress是基于GPL协议的。GPL协议最主要的几个原则：1、确保软件自始至终都以开放源代码形式发布，保护开发成果不被窃取用作商业发售。任何一套软件，只要其中使用了受 GPL 协议保护的第三方软件的源程序，并向非开发人员发布时，软件本身也就自动成为受 GPL 保护并且约束的实体。也就是说，此时它必须开放源代码。2、GPL 大致就是一个左侧版权（Copyleft，或译为“反版权”、“版权属左”、“版权所无”、“版责”等）的体现。你可以去掉所有原作的版权 信息，只要你保持开源，并且随源代码、二进制版附上 GPL 的许可证就行，让后人可以很明确地得知此软件的授权信息。GPL 精髓就是，只要使软件在完整开源 的情况下，尽可能使使用者得到自由发挥的空间，使软件得到更快更好的发展。3、无论软件以何种形式发布，都必须同时附上源代码。例如在 Web 上提供下载，就必须在二进制版本（如果有的话）下载的同一个页面，清楚地提供源代码下载的链接。如果以光盘形式发布，就必须同时附上源文件的光盘。4、开发或维护遵循 GPL 协议开发的软件的公司或个人，可以对使用者收取一定的服务费用。但还是一句老话——必须无偿提供软件的完整源代码，不得将源代码与服务做捆绑或任何变相捆绑销售。没有违反原则就没有错。------------------------------根据 @破宝 提出的意见以及引用 @maxSonic的评论，描述大致如下：1.凡是GPL的代码，只要你商业发布为产品，别人都有权跟你索要源码。但是你并没有主动披露源代码的义务。2.当你分发这样的程序，无论免费或收费，你都必须向接收方授予与你相同的自由权利，你必须确保接收方同时接收源代码，或者能取得源代码，你必须展示这里的条款使接收方知道他们的权利。此处的接收方仅指个人或组织，一台机器（“商业可用机器”）是不应该被视作接收方的。3.如果你用到了WordPress及其演绎的产品并发布为商业作品，但是接受方是一台机器（“商业可用机器”），那么第二条将不适用，发布方没有义务向公众提供源代码，除非将全站作为文件打包提供下载。关于APGL，引用@破宝的话使用GPL程序的网站没有义务向公众公开源代码，这个“漏洞”被称作ASP Loophole。为避免这一点，后来还专门出了个叫做Affero GPL的许可证Affero General Public License 。在AGPL的开篇这样写着：A secondary benefit of defending all users' freedom is that improvements made in alternate versions of the program, if they receive widespread use, become available for other developers to incorporate. ... However, in the case of software used on network servers, this result may fail to come about. The GNU General Public License permits making a modified version and letting the public access it on a server without ever releasing its source code to the public.保卫所有用户的自由权利（这里指自由复制/分发/修改的权利）的第二点好处是，当有更多人使用时，分支版本中出现的优化升级能够被其他开发者所吸收利用。…… 可是，当软件被用在网络服务器时，这个好处就无效了，因为GPL许可证允许修改形成新版本，并在不对外发布源代码的情况下，就部署到服务器上让公众访问。The GNU Affero General Public License is designed specifically to ensure that, in such cases, the modified source code becomes available to the community. It requires the operator of a network server to provide the source code of the modified version running there to the users of that server. Therefore, public use of a modified version, on a publicly accessible server, gives the public access to the source code of the modified version. 因此AGPL许可证的设计目标特别针对这种情况，使修改版本的源代码也能被社区所用。AGPL许可证要求网络服务器的管理方，必须对它的用户公开它所运行的修改版本的源代码。也就是说，在公众可访问的服务器上、为公众提供服务的修改版本，需要对公众开放其源代码。

第一点：正是有了那么多的开源程序，网站建设，特别是普通的公司型网站建设的价格才会变得那么便宜，几千上万块可以做一个内容发布型的公司网站。如果从头开始开发，那做一个公司网站的价格，恐怕得10w往上加才行。而且安全和速度远远比不上Wordpress，因为Wordpress是集合了世界上成千上万的开发者的智慧。第二点：Wordpress是全世界最流行的CMS程序。虽然它的初衷是一个博客程序，但由于其开放性以及世界众多开发者对插件库的贡献，它已经不仅仅是博客程序，通过二次开发，几乎可以胜任任何类型的网站。国外使用Wordpress的大公司不计其数，包括众多权威政府机关。因此，这个老板的公司网站用Wordpress开发，一点都不失礼，反而这才是高端大气上档次（看看国内那些dede、phpcms之类的网站程序的后台，再看看Wordpress的后台就知道了）。第三点：作为内容发布型的公司网站，最核心的部分其实不是功能，而是外观。Wordpress只是个提供发布功能的程序，真正体现网站建设者价值的是网站的外观设计和实现，而这方面，这个老板是很满意的。至于网站是用什么开发的，根本不应该在意。如果发现网站是基于现有的模板做的，而且外观上改动的地方不多，这才是程序员怠工的体现，才应该恼羞成怒。第四点：Wordpress版权声明是允许去掉的，而且一般应该去掉，毕竟跟客户网站是没有关系的，放在底部总是有点影响。不过国外很多提供下载的Wordpress模板都会习惯性的带有版权信息，如果自己写模板则很少会带上这个，从这点倒是可能怀疑程序员是用了现成模板。

Wordpress 的好处是实现基本的需求（包括做的炫一些）很容易，特别是在用现成模板的情况下。熟练的话花一小会时间就能搭出一套，然后用几十美元买个现成的模板。不少人觉得 Wordpress 搭网站简单便宜，就是因为如此。但是实际上，如果要做出原创的 Wordpress 主题，特别是要做好，并不是那么简单的事。Wordpress 仅仅帮你节省了框架和后台方面的力气，实际上即使不用 Wordpress，也可能会用别的开源组件之类。不需要从自己造轮子开始造车。所以归根结底是否用 Wordpress 做并不是关键，关键是做的东西是否是自己原创的，还是只是简单的做了二道贩子。
